## 芽芽助教项目背景

芽芽助教是一个面向 K12 全学段、多学科场景的自我进化 AI 教学适配引擎。它以真实教材知识图谱为骨架、以学习行为为真值、以错因诊断为核心、以策略调度为动作系统、以成长记忆为长期资产、以个性化课堂生成为输出、以课堂内容自我进化为生命线。

一句话：孩子用得越久，芽芽越懂他；芽芽教得越久，越知道怎么教他。

**当前建设范围**：MVP 阶段为人教版小学数学 5–6 年级，第二阶段扩展至 1–4 年级。

**数据底座**：
- 394 个知识点（G1-G4: 205 个，G5-G6: 189 个），全量录入
- 86 个错因（L1 通用 10 个 + L2 数学特有 76 个），全量定义
- 1390 条知识点-错因绑定，394 个 KP 全覆盖
- 两层知识图谱（学习路径层 + 诊断依赖层），G1-G6 全部建成
- 题型规格矩阵（260KB）
- 数据质量门禁 data_quality_gate.py（9 铁规全覆盖）

**引擎架构**：诊断引擎（四步：初筛→匹配→验证→溯源）→ 策略引擎（六态调度）→ 个性化课堂生成引擎（本文档）→ 家长摘要（中性安全反馈）→ 质量仪表盘（24 指标）。

**文档集**：18 份规划文档（01-18）。

## 本文档背景

这是芽芽助教产品文档集的第 18 号文件，定义个性化课堂的生成策略——从 Phase 1 的方案 C（骨架预设+AI 血肉），到 Phase 2 的方案 C+D（骨架+积木拼装），到 Phase 3+ 的方案 E（自我进化的教学生命体）。它回答「策略引擎选好了策略包之后，怎么根据每个学生的画像，生成一堂只属于他的互动课堂」——是芽芽助教「千人千面」能力的核心输出层。

## 一、目标

个性化课堂生成引擎的目标不是「把教案换个颜色」，而是让每个学生感受到「这堂课就是为我准备的」。

一堂课由五个环节组成：开场破冰 → 概念讲解 → 互动练习 → 反馈话术 → 收尾总结。每个环节都有个性化钩子——同一个知识点「分数加减法」，对不同的学生，开场方式不同、用的例子不同、练习的梯度不同、做对后的表扬方式不同、结束时说的话不同。

核心原则：**骨架保证教学正确性（不会教错），血肉提供个性化表达（让孩子觉得被理解），进化让课堂越上越好（越教越懂怎么教他）。**

## 二、三个阶段：C → D → E

### 2.1 方案 C（骨架预设 + AI 血肉）—— Phase 1 MVP

**一句话比喻**：私厨定好菜谱框架（骨架），现场调味（AI 血肉）你来定。

具体做法：课堂结构拆成固定的五环节骨架——每个环节的「必须做什么」由规则引擎决定。但环节内的具体内容——用什么生活例子、说什么鼓励话术、练习题的场景怎么选——由 LLM 根据学生画像实时生成。

骨架按知识点类型分组，不到 10 个模板覆盖全部知识点：

| 骨架类型 | 适用场景 | 五环节编排 |
|---------|---------|-----------|
| 计算类 | 小数乘除法、分数加减、百分数计算等 | 生活场景引入 → 步骤拆解示范 → 跟练（一模一样的题）→ 变式练习 → 纠错 → 独立挑战 |
| 概念类 | 因数倍数、分数意义、比例、方程等 | 生活场景引入 → 抽象提炼概念 → 互动判断"是不是"→ 例题巩固 → 总结 |
| 应用题类 | 行程问题、工程问题、鸡兔同笼等 | 读题（审题侦探）→ 画图建模 → 列式求解 → 变情境练习 → 自编题挑战 |
| 几何类 | 面积、体积、圆、长方体等 | 动画演示推导 → 公式记忆锚点 → 基础代公式 → 变图形练习 → 逆向（给面积求边长） |
| 单位换算类 | 长度/面积/体积/时间单位换算 | 生活对比（一厘米多长→一把尺子）→ 桥梁法示范 → 正向换算 → 反向换算 → 混合换算 |
| 统计类 | 折线图、条形图、扇形图 | 读图示范 → 提取信息练习 → 制图练习 → 判断误导图表 → 总结 |
| 复习类 | 混合复习、期末复习 | 快速回顾（记忆唤醒）→ 错题重练 → 新题混入 → 自我评估 |

骨架模板定义每个环节的「必须包含什么」（例如计算类骨架的「步骤拆解」环节必须包含：明确每一步的输入和输出、标注易错点、做一步让学生验证一步），但不限定具体用什么数字、什么场景、什么话术。

AI 血肉填入的内容包括：
- 生活例子的场景选择（来自学生偏好：喜欢游戏→用游戏装备材料分配；喜欢美食→用分披萨/分糖果）
- 讲解节奏的长度（来自行为模式：慢且稳型→每步间停顿；快且粗心型→每步后加确认）
- 鼓励话术的风格（来自激励偏好：喜欢积分→"这步对了，+5分！"；喜欢进步→"比上次快多了！"）
- 纠错语言的针对性（来自错因历史：上次忘了通分→"先看看分母，上次是不是在这里卡过？"）
- 练习题的场景包装（来自行为模式：快但粗心→穿插审题提示；连错退出型→第一题特别简单建立信心）

回答「千人千面怎么实现」：
- 千面：每个学生因为画像不同，同一个知识点看到的生活例子不同、话术不同、节奏不同
- 千人：画像系统（15/16/17 号）让引擎知道每个学生的偏好和状态，骨架保证教学内容在教材边界内

### 2.2 方案 D（方案 C + 积木拼装）—— Phase 2

**一句话比喻**：AI 做过的菜，好吃的留下来做成预制调料包，下次直接组合。

在方案 C 运行一段时间后，系统积累了大量 AI 生成的血肉内容。方案 D 的核心动作是「沉淀」——把效果好的血肉从一次性的 AI 输出变成可复用的积木块。

积木块定义：最小可独立使用的教学片段。例如：
- 「通分步骤图」积木——30 秒动画，展示找公分母→分子调整→相加→约分
- 「披萨分法」积木——用披萨场景解释分数加减，适合偏好美食的学生
- 「审题三步提醒」积木——快且粗心型学生的答题前确认步骤
- 「进步对比卡」积木——对比上次正确率，适合偏好成长可见的学生

每个积木块有标签体系：
- 知识点标签：适用于哪些 KP
- 骨架标签：用于哪个环节（开场/讲解/练习/反馈/收尾）
- 画像标签：最适合的学生类型（偏好维度、行为模式、年级分段）
- 效果标签：对该标签学生的历史正确率变化、反应时间变化

积木来源：AI 在方案 C 中生成的血肉，如果该堂课该环节的 effectScore ≥ 1（有帮助），则该血肉自动进入「积木候选池」。连续 3 次 effectScore ≥ 1 后，升级为正式积木块。

拼装算法：骨架决定环节顺序和必需要素 → 画像系统提供学生标签 → 算法从积木库中按标签匹配和效果排序选取最优积木 → 拼接成一堂课。

### 2.3 方案 E（自我进化的教学生命体）—— Phase 3+

**一句话比喻**：不是人教系统做菜，是系统自己从每个食客的反应中学会怎么做他最爱吃的。

方案 E 不是推翻 C 和 D，而是在 D 的基础上增加了两层自我进化能力：

**第一层：积木本身会变异**

系统发现某个积木块（如「通分步骤图」）对某类学生效果持续下降（effectScore 连续衰减），自动触发变异：
- 视觉素材替换：如果学生对「披萨图」反应比「数线图」快，系统自动把通分步骤图里的视觉素材从数线替换为披萨风格
- 节奏调整：如果学生在第三步（分子调整）停顿最长，系统自动在该步插入一个中间步骤「确认分母是否相同」
- 话术变体：系统自动生成同一积木的多个话术版本，根据学生反应选择最优版本

**第二层：拼装规则从人工写变为系统自学**

方案 C 和 D 的拼装规则是人写的（如「对快且粗心型插审题提醒」）。方案 E 中，系统自己发现规律——不需要任何教育理论预设：
- 发现：某学生在周五下午状态差时，用「挑战闯关」反而比「舒缓讲解」更能拉回注意力
- 这不是任何教育理论会说的规则，是这个孩子自己的规律
- 系统学会后，周五下午的课自动切换节奏，不再走预设的「能量低→舒缓」逻辑

进化信号采集：
- 每堂课的每个环节，记录学生的反应：看题时间变化、停顿位置、答题正确率、是否求助、是否退出
- 好的信号（反应快+正确率高+无退出倾向）→ 该环节的「基因」被标记为优势
- 差的信号（停顿长+错误+退出）→ 该环节的「基因」被标记为劣势
- 基因存活规则：连续 3 次 effectScore ≥ 1 的保留，连续 2 次 ≤ 0 的进入观察期，连续 3 次 ≤ 0 的淘汰

方案的阶段递进关系：
- Phase 1：纯方案 C（骨架+AI 血肉），种子阶段，重点在「能生成」
- Phase 2：方案 C+D（骨架+积木拼装），沉淀阶段，重点在「能复用」
- Phase 3+：方案 E（自我进化），进化阶段，重点在「能自成长」

## 三、课堂五环节结构

每堂个性化课由五个环节组成。每个环节都有骨架约束（保证正确）和个性化钩子（千人千面）。

### 3.1 环节一：开场破冰

**骨架约束**：必须完成（1）打招呼（2）激活前置知识（如果前置知识点掌握度 < 0.6，先温习）。（3）告知本堂课目标。

**个性化钩子**：
| 画像输入 | 开场方式 |
|---------|---------|
| 偏好激励 = points（积分） | "今天挑战完成，解锁三颗星！" |
| 偏好激励 = growth_visible（看进步） | "上次正确率 70%，今天咱们破 80%，有信心吗？" |
| 今天状态 = easy（轻松点） | "今天慢慢来，咱们不赶时间" |
| 今天状态 = challenge（挑战） | "今天有一道隐藏关卡，做对才能见到" |
| 上节课退出（pendingAction=energy_recovery） | 不说分数，轻松开场："今天换个讲法，试试新的" |
| 行为模式 = fast_careless（快但粗心） | "今天芽芽教你一个绝招——放慢一步，对更多" |
| 年级 1-3 | 动画芽芽形象+语音打招呼 |
| 年级 4-6 | 文字+emoji 开场 |

### 3.2 环节二：概念讲解

**骨架约束**：必须包含（1）教材定义（2）一个生活化类比或图示（3）至少一个正例和一个反例让判断（4）关键公式/步骤显式展示。

**个性化钩子**：
| 画像输入 | 讲解方式 |
|---------|---------|
| 呈现偏好 = animation（动画） | 优先动画演示 |
| 呈现偏好 = graphic（图文） | 图文分步展示 |
| 呈现偏好 = audio（语音）或 年级 1-3 | 语音讲解+跟读 |
| 呈现偏好 = practice_first（直接做题） | 讲解缩短为 30 秒核心摘要，快速进入练习 |
| 主错因 = concept_confusion（概念混淆） | 正反例对比强调，加"这个概念不是……" |
| 主错因 = calculation_error（计算失误） | 讲解中嵌计算检查点 |
| 知识难度 = A 级（了解） | 1 个生活例子，不讲推导 |
| 知识难度 = D 级（运用） | 2 个不同场景例子+推导过程 |

### 3.3 环节三：互动练习

**骨架约束**：必须包含（1）一道与示范同结构的模仿题（2）一道变式题（3）一道独立挑战题。题目从教材区域内出，不超纲。

**个性化钩子**：
| 画像输入 | 练习配置 |
|---------|---------|
| 挑战偏好 = easy（简单） | 题量减半，前两道超基础 |
| 挑战偏好 = challenge（挑战） | 加变式题，第三道跳级难度 |
| 行为模式 = slow_stuck（慢且卡住） | 每道题之前给一个"思路提示"，不直接给答案 |
| 行为模式 = fast_careless（快但粗心） | 每道题提交前弹"再看一遍"确认步骤 |
| 行为模式 = error_quitter（连错退出型） | 第一题难度调到「肯定能做对」级别，建立信心 |
| helpSeekingStyle = self_solve（自己想） | 延迟提示按钮出现时间，让孩子先独立思考 |
| helpSeekingStyle = hint_dependent（依赖提示） | 提示从 level 1 开始，循序渐进，不跳级 |
| 策略包 = 分步引航包 SP_G_B01 | 把一道题拆成 3 个小步骤，做完一步验一步 |
| 策略包 = 精准计算包 SP_G_C05 | 仅出纯计算题，无文字应用题干扰 |

### 3.4 环节四：反馈话术

**骨架约束**：每道题做完必须有反馈。错误时必须给出具体错因指向（不能说「再想想」），正确时必须给具体肯定（不能说「你真棒」）。

**个性化钩子**：
| 画像输入 | 反馈方式 |
|---------|---------|
| 激励偏好 = praise（夸他） | 具体表扬："你主动检查了分母，这个习惯特别好！" |
| 激励偏好 = points（积分） | "+3 分！因为你在第三步特别细心" |
| 激励偏好 = growth_visible（看进步） | "看，上次这道题你做错了，这次对了。进步了！" |
| 激励偏好 = unlock（解锁） | "又过一关！离隐藏关卡还差 1 道题" |
| 行为模式 = error_persister（连错坚持型） | 做错时先肯定："你已经试了三次了，芽芽看到你在努力。来，咱们换个角度——" |
| 行为模式 = error_quitter（连错退出型） | 做错时不强调错误，说"这道题的陷阱藏得特别深，芽芽第一次也错过了，咱们一起看看" |
| 年级 1-3 | 语音反馈+动画表情 |
| 年级 4-6 | 文字反馈+emoji |

### 3.5 环节五：收尾总结

**骨架约束**：必须包含（1）今天学了什么（2）对了多少（3）下次预告。

**个性化钩子**：
| 画像输入 | 收尾方式 |
|---------|---------|
| 学习能量低或触发能量恢复包 | 轻松收尾："今天到这里，你很努力了。保存体力，下次再战。" |
| 完成率高+能量正常 | 预告挑战："下次我们挑战一个更难的变化，你准备好了吗？" |
| parentGoal = foundation（补基础） | 提示"今天补了一个基础，下次继续加固" |
| parentGoal = boost（提分突破） | 强调"今天比上次多对了 1 道，突破在发生" |
| 年级 1-3 | 语音+动画芽芽说再见 |
| 年级 4-6 | 文字总结+emoji |

## 四、输入接口

个性化课堂生成引擎需要三组输入：

### 4.1 教什么（来自数据底座）

| 输入字段 | 来源 | 用途 |
|---------|------|------|
| knowledgeCode | knowledge.json | 确定是哪个知识点 |
| name | knowledge.json | 知识点名称 |
| curriculumLevel | knowledge.json | A/B/C/D 决定讲解深度和练习题难度范围 |
| teachingGoal | knowledge.json | 教参教学目标，约束教学边界 |
| keyPoints | knowledge.json | 教学重点，必须覆盖 |
| commonMistakes | knowledge.json | 常见错误，讲解时预判 |
| textbookPage | knowledge.json | 教材页码范围，约束讲解不超纲 |
| questionType bindings | question_type_matrix.json | 该出什么题型 |
| 前置知识点 | knowledge_graph.json | 如需温习前置知识，从低年级调内容 |

### 4.2 给谁上（来自画像系统 15/16/17 号）

| 输入字段 | 来源 | 用途 |
|---------|------|------|

| helpSeekingStyle | 15 号问题 2 / 17 号 L1 | 给提示/给例子/一步步带/自己想 |
| challengePreference | 15 号问题 3 | 简单/刚好/挑战/隐藏关卡 |
| motivationPreference | 15 号问题 4 / 17 号 L1 | 夸他/积分/看进步/解锁 |
| todayState | 15 号问题 5 / 课前状态卡 | 轻松/挑战/自己先想/AI同学陪（仅本次课） |
| parentGoal | 15 号问题 6 | 补基础/稳成绩/提分/兴趣/超前 |
| parentPace | 15 号问题 7 | 稳一点/同步/专攻弱点/挑战拔高 |
| behaviorPattern | 16 号 §三（10 类） | 快且准/快但粗心/慢但稳/慢且卡住/高频求助/拒绝提示/连错退出/连错坚持 |

## 六、Phase 1 实现范围（方案 C）

Phase 1 实现以下内容：

- [ ] 定义 7 种骨架模板（计算类/概念类/应用题类/几何类/单位换算类/统计类/复习类）
- [ ] 实现骨架模板引擎——给定知识点+画像输入+策略处方，输出五环节骨架结构
- [ ] 实现 AI 血肉生成器——调用 LLM（接入 01 号 §3 的 MCP 化路径），根据骨架+画像生成具体内容
- [ ] AI 生成内容缓存——同一学生+同一知识点+同类画像组合，缓存 24 小时内不重复调用 LLM
- [ ] AI 生成内容质检——包含：教材边界检查（不含超纲概念）、禁止词检查（无 AI 字眼）、学生适配检查（话术级别匹配年级）
- [ ] 输出结构化课堂脚本 JSON（见第五节 Schema）
- [ ] 课堂执行后记录 effectSignal——为方案 D/E 积累进化数据

Phase 1 不实现：
- 积木沉淀系统（方案 D，Phase 2）
- 积木变异和拼装规则自学（方案 E，Phase 3+）
- 课堂脚本的自进化闭环（方案 E，Phase 3+）

## 十、验收标准

### Phase 1（方案 C）验收：

- [ ] 7 种骨架模板覆盖 394 个知识点（每种知识点能匹配到一个骨架类型）
- [ ] lesson_started 后 3 秒内生成课堂脚本 JSON（含缓存命中；首次 LLM 生成 ≤ 10 秒）
- [ ] 生成内容不含教材外的新概念（教材边界检查通过率 100%）
- [ ] 生成内容不含「AI」「人工智能」等禁止词
- [ ] 不同偏好的学生看同一知识点，生成的生活例子不同（千人千面验证：用两组完全不同画像的学生，diff 课堂脚本，核心差异项 ≥ 3 个）
- [ ] 同一个学生连续两次看同一知识点，第二堂课引用了上一次的错因历史
- [ ] 低年级（1-3）课堂使用语音引导，高年级（4-6）使用图文引导
- [ ] 课堂脚本 JSON 格式校验通过（符合第五节 Schema）

### Phase 2（方案 C+D）验收：

- [ ] 积木库 ≥ 50 个正式积木块（覆盖所有 7 种骨架类型）
- [ ] 积木拼装课堂 > 30% 的环节来自积木而非 LLM 生成（LLM 调用量下降 30%）
- [ ] 积木拼装课堂的骨架完整性与方案 C 一致（不因积木化而漏环节）

### Phase 3+（方案 E）验收：

- [ ] 至少 1 个积木块发生了成功的自动变异（变异后的 effectScore 高于变异前）
- [ ] 至少发现 1 条人工规则未覆盖、系统自学的个性化拼装规则
- [ ] 自进化后的课堂平均 effectScore 高于进化前（统计显著性 p < 0.05）
