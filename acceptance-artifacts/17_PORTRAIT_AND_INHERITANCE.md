## 芽芽助教项目背景

芽芽助教是一个面向 K12 全学段、多学科场景的自我进化 AI 教学适配引擎。它以真实教材知识图谱为骨架、以学习行为为真值、以错因诊断为核心、以策略调度为动作系统、以成长记忆为长期资产。

一句话：孩子用得越久，芽芽越懂他；芽芽教得越久，越知道怎么教他。

**当前建设范围**：MVP 阶段为人教版小学数学 5–6 年级，第二阶段扩展至 1–4 年级。

**数据底座**：
- 394 个知识点（G1-G4: 205 个，G5-G6: 189 个），全量录入
- 86 个错因（L1 通用 10 个 + L2 数学特有 76 个），全量定义
- 1390 条知识点-错因绑定，394 个 KP 全覆盖
- 两层知识图谱（学习路径层 + 诊断依赖层），G1-G6 全部建成
- 题型规格矩阵（260KB）
- 数据质量门禁 data_quality_gate.py（9 铁规全覆盖）

**引擎架构**：诊断引擎（四步：初筛→匹配→验证→溯源）→ 策略引擎（六态调度）→ 家长摘要（中性安全反馈）→ 质量仪表盘（24 指标）。

**文档集**：17 份规划文档（01-17）。

## 本文档背景

这是芽芽助教产品文档集的第 17 号文件，定义四层画像结构（L1 全局学生画像 / L2 学科画像 / L3 知识点画像 / L4 题型画像）、画像间的信息传导协议和学段跃迁时画像数据的继承规则。它回答「系统怎么记住一个学生、怎么在不同学科和学段之间传递认知」——对应 PRD 模块九「成长记忆与画像继承模块」和 PRD §16.2-16.5。

## 一、目标

画像系统的目标不是给学生「打分」，而是让引擎在每次教学决策时拥有足够的上下文。四层画像从粗到细，逐层精确——全局层给出「这个人怎么学」的总体判断，题型层给出「这道题他会不会碰壁」的具体判断。

核心原则：**画像由行为驱动，不由问卷锁定。跨学科不传结论，只传关注度。学段跃迁时该忘的忘，该留的留。**

## 二、四层画像结构

### 2.1 L1 全局学生画像

最顶层，记录跨学科、跨学段的稳定行为特质。同一学生的所有学科共享此层。

| 记录内容 | 字段 | 说明 |
|---------|------|------|
| 全局行为特质 | behaviorTraits | 学习节奏偏好（快/中/慢）、坚持性（退出型/坚持型）、冲动倾向（快但错多/深思慢稳） |
| 激励偏好 | motivationPreference | praise / points / growth_visible / unlock（来自 15 号问题 4，由行为修正） |
| 求助习惯 | helpSeekingStyle | small_hint / similar_example / step_guidance / self_solve / hint_dependent / hint_avoidant |
| 呈现偏好 | presentationPreference | animation / graphic / audio / practice_first |
| 全局错因倾向 | globalErrorTendency | 跨学科的稳定错因模式（如「容易跳步」「需要多次验证」） |
| 专注力基线 | focusBaseline | 平均有效学习时长、中断模式 |
| 全局历史快照 | snapshots | 关键节点的画像快照（学年结束、学段跃迁前） |

### 2.2 L2 学科画像

每个学科独立维护一份，记录该学科内的学习特征。

| 记录内容 | 字段 | 说明 |
|---------|------|------|
| 学科掌握度概览 | subjectMastery | 学科整体掌握水平（0-1），由 L3 知识点画像加权汇聚 |
| 学科错因倾向 | subjectErrorTendency | 该学科内高频错因分布（如数学=计算失误 40%、概念混淆 30%） |
| 学科策略效能 | subjectStrategyEfficacy | 该学科内各策略包的历史有效率 |
| 学科学习目标 | subjectGoal | 来自家长目标（foundation / sync / boost / interest / advanced，见 15 号 §3.2） |
| 学科行为基线 | subjectBehaviorBaseline | 该学科的平均答题时间、正确率、完成率 |

### 2.3 L3 知识点画像

每个知识点（KP）独立维护一份，是诊断引擎最重要的输入层。

| 记录内容 | 字段 | 说明 |
|---------|------|------|
| 知识点掌握度 | masteryScore | 0-1，引擎综合计算（正确率 + 答题时间 + 提示依赖 + 迁移表现） |
| 错因假设 | errorHypotheses | 该知识点下的概率化错因向量（来自 diagnosis_updated 累积） |
| 复习状态 | reviewStatus | 最后一次复习时间、间隔天数、需要复习的紧迫度 |
| 迁移能力 | transferAbility | 该知识点在不同题型/场景下的表现差异 |

### 2.4 L4 题型画像

最细粒度，每个知识点 × 每种题型独立维护。

| 记录内容 | 字段 | 说明 |
|---------|------|------|
| 题型熟悉度 | typeFamiliarity | 0-1，该知识点该题型下的正确率 |
| 成功率 | successRate | 最近 N 次该题型下的正确率 |
| 常见错因 | typeCommonErrors | 该题型下高频错因 |
| 推荐策略 | typeRecommendedStrategies | 对该题型有效的策略包 |
| 考试/课标权重 | examWeight | 该题型在考试中的重要性（0-1） |

## 三、信息传导协议

画像层之间不是孤立的——信息从细粒度层向上汇聚，也从粗粒度层向下传导。传导遵循「信号传递、不做结论复制」原则。

### 3.1 L3 → L2：知识点画像汇聚为学科画像

汇聚规则：
- 学科掌握度 = Σ(L3.masteryScore × KP权重) / Σ(KP权重)
- KP权重 = curriculumLevel 映射（A=1, B=1.5, C=1, D=2，反映教学重要性和难度）
- 学科错因倾向 = 各 L3 错因概率按知识点加权平均
- 汇聚频率：每次 L3 更新后当天批量重算（非实时，避免计算压力）

### 3.2 L2 ⇄ L2：跨学科传导

规则：
- **不传导结论**：数学「计算失误概率 0.8」不会直接告诉语文「这个学生粗心」
- **只传导关注度**：高置信度学科可以向低置信度学科传递「请注意这个方面」
- **传导强度 < 1**：默认不超过 0.7。即源学科的 1.0 关注信号，传导到目标学科最多 0.7
- **低置信度学科不传导**：源学科的学科掌握度 < 0.3（数据太少）时不传导
- **目标学科必须独立验证**：收到传导后，目标学科在该方向加轻量观察，不直接修改错因概率
- **目标学科表现良好时自动衰减**：如果目标学科在传导方向上表现正常（连续 30 天无异常），传导影响衰减至 0

例子：
- 数学发现该学生「读题经常遗漏关键信息」→ 向语文学科传导关注度 0.5 → 语文引擎在信息筛选能力上增加观察权重 → 如果语文表现正常，30 天后衰减至 0
- 这是 PRD §14.5 跨学科错因传导的具体实现

### 3.3 L2 → L1：多个学科抽象为全局行为特质

汇聚规则：
- 全局行为特质 = 各学科行为基线的聚合（取中位数，避免单一学科偏差）
- 全局错因倾向 = 各学科错因分布的交叉项（如 3 个以上学科均高频出现 compehension_bias → 提升到全局层）
- 专注力基线 = 所有学科有效学习时长的加权平均

### 3.4 L1 → L2：全局特质作为学科冷启动先验

传导规则：
- 新学科首次进入时，L1 全局画像作为先验注入
- 先验权重：初始 0.3（行为积累前）= 30% 依赖全局判断，70% 等待新学科数据
- 衰减：新学科每积累 10 次有效课堂，先验权重 -0.05，最低降至 0.05
- 仅传导：行为特质、激励偏好、求助习惯、呈现偏好
- 不传导：错因概率、策略效能（这些都是学科特定的）

## 四、学段跃迁继承规则

学生从低学段升入高学段时（如小学升初中、初中升高中），画像数据面临「该留什么、该忘什么」的问题。

### 4.1 完全继承（full_inherit）

以下画像维度跨学段完全继承，不衰减：

| 维度 | 原因 |
|------|------|
| 专注力模式 | 不随学科和年级变化，是稳定的个人特质 |
| 激励偏好 | 不随年级变化（喜欢表扬的孩子到高中可能仍然喜欢表扬） |
| 求助依赖度 | 求助习惯是长期养成的学习策略 |
| 冲动指数 | 快但错多的倾向是稳定的认知风格 |
| 呈现偏好 | 不随年级变化（喜欢动画的学生在低年级和高年级都偏好动画） |

### 4.2 衰减继承（decay_inherit）

以下维度继承但按衰减系数打折：

| 维度 | 衰减系数 | 原因 |
|------|---------|------|
| 学科关联能力 | 0.6 | 学段变了，知识结构重构，旧关联仅部分有效 |
| 错因倾向 | 0.4 | 不同学段的错因分布可能不同（如小学「单位混淆」到初中自动消失） |
| 题型倾向 | 0.5 | 题型体系随学段改变，旧题型偏好参考价值有限 |

### 4.3 完全清零（clear）

以下维度不继承，新学段从零开始：

| 维度 | 原因 |
|------|------|
| 具体知识点掌握度 | 学段变了，旧知识点不直接等价于新知识点 |
| 策略包效能 | 每种策略在不同学段的效果不同，不能沿用 |
| 学科掌握度 | 同学科在新学段是全新的内容体系 |
| L4 题型画像 | 题型体系彻底改变 |

### 4.4 继承范围

继承仅在同一学生、同一学科内发生。不同学科之间不走学段跃迁继承——只通过 §3.2 的跨学科传导协议交换信号。

### 4.5 跃迁触发

当系统检测到 `grade` 字段发生跨学段变化时自动触发：
- 小学升初中（grade 6 → 7）
- 初中升高中（grade 9 → 10）

年级内变化（如 5→6）不触发学段跃迁继承，画像数据保持完整连续。

## 五、校准期

学段跃迁后，系统进入校准期。

### 5.1 校准期时长

- 小学升初中：前 15 节有效课堂
- 初中升高中：前 15 节有效课堂

### 5.2 校准期内规则

| 规则 | 说明 |
|------|------|
| 强策略暂停 | 冷启动保护：所有强策略（变式特训、错因专项）暂停，仅用轻量/中度策略 |
| 诊断置信度折扣 | 诊断引擎的初始假设置信度 × 0.7（因为基于衰减继承的先验可能不够准） |
| 行为观测权重提高 | 校准期内每道题的行为信号权重 × 1.5（加速重建新学段的行为基线） |
| 家长摘要标注 | 家长摘要中加入「新学段适应期」提示 |

### 5.3 校准期结束条件

- 完成 15 节有效课堂
- 且至少覆盖 3 个不同知识点
- 且最近 3 节课行为基线稳定（正确率方差 < 0.15）

满足后，校准期自动结束，画像恢复标准置信度，强策略恢复可用。

## 六、画像数据生命周期

### 6.1 冷启动

新学生首次进入：
- 使用 15 号文档的偏好问卷初始化 L1 部分维度
- L2/L3/L4 完全空白
- 诊断引擎使用统一先验概率（0.5）
- 策略引擎仅用轻量策略

### 6.2 日常更新

- L4 题型画像：每次 answer_submitted 后实时更新
- L3 知识点画像：每次 diagnosis_updated 后批量更新
- L2 学科画像：每天凌晨批量重算（非实时）
- L1 全局画像：每周批量重算（非实时）

### 6.3 数据保留

- 所有画像数据保留至学生账号删除
- 家长撤回授权后，画像数据暂停更新但不删除（保留在后台，再次授权后恢复）
- 阶段画像快照在每学期结束时生成一份，作为长期历史参照

### 6.4 数据清理

以下情况触发画像数据清理：
- 家长主动删除账号 → 全部画像数据 30 天后物理删除
- 家长撤回授权 → 画像数据冻结、暂停更新，保留 180 天，超期物理删除
