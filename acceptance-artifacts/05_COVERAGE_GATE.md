# 05 CoverageGate 覆盖门禁

## 芽芽助教项目背景

芽芽助教是一个面向 K12 全学段、多学科场景的自我进化 AI 教学适配引擎。它以真实教材知识图谱为骨架、以学习行为为真值、以错因诊断为核心、以策略调度为动作系统、以成长记忆为长期资产。

一句话：孩子用得越久，芽芽越懂他；芽芽教得越久，越知道怎么教他。

**当前建设范围**：人教版小学数学 1-6 年级全量覆盖。

**数据底座**：
- 394 个知识点（G1-G4: 205 个，G5-G6: 189 个），全量录入
- 86 个错因（L1 通用 10 个 + L2 数学特有 76 个），全量定义
- 1390 条知识点-错因绑定，394 个 KP 全覆盖
- 两层知识图谱（学习路径层 + 诊断依赖层），G1-G6 全部建成
- 题型规格矩阵（260KB）
- 数据质量门禁 data_quality_gate.py（9 铁规全覆盖）

**引擎架构**：诊断引擎（四步：初筛→匹配→验证→溯源）→ 策略引擎（六态调度）→ 家长摘要（中性安全反馈）→ 质量仪表盘（24 指标）。

**文档集**：14 份规划文档（01-14），覆盖产品愿景、开发路线、教材来源、分级规则、覆盖门禁、知识图谱、错因分类、策略包、事件协议、数据模型、诊断引擎、策略引擎、家长摘要、质量仪表盘。

## 本文档背景

这是芽芽助教产品文档集的第 05 号文件，定义知识点、题型、错因、策略包、家长摘要的上线门禁规则。它回答「能不能发布」——例如 A 级知识点必须有教材来源、至少绑定 3 个错因和 2 个策略包，缺任意一项不允许发布。本文档的规则已实现为 data_quality_gate.py 自动化脚本，运行后 9/9 PASS 表示数据底座健康。

## 定义

CoverageGate 是正式发布门禁系统，用来判断知识点、题型、错因、策略、验证动作、家长摘要是否具备上线条件。

CoverageGate 规则由人类确认，AI 可以辅助生成脚本实现、测试用例、mock 数据和缺项修复建议，但不能自行降低发布标准或绕过门禁。

## KnowledgeNode 门禁

每个知识点必须检查：

1. `knowledgeCode`
2. 年级
3. 册别
4. 单元
5. 小节 / 课时
6. 教材来源
7. 前置知识
8. 后续知识
9. A/B/C 分级
10. 题型
11. 错因
12. 验证动作
13. 推荐策略
14. 掌握度规则
15. 练习题
16. 验证题
17. 家长摘要模板

## A 级知识点门禁

A 级知识点必须满足：

```text
必须有教材来源
必须有前置知识
必须有后续知识
必须至少绑定 2 个题型
必须至少绑定 3 个常见错因
必须至少绑定 2 个策略包
必须至少有 1 个验证动作
必须至少有 5 道基础题
必须至少有 5 道变式题
必须至少有 3 道错因验证题
必须有家长摘要模板
必须有掌握度规则
```

缺任意一项，不允许发布。

## B 级知识点门禁

```text
必须有教材来源
必须有前置/后续关系
必须至少绑定 1 个题型
必须至少绑定 1 个常见错因
必须至少绑定 1 个策略包
必须至少有 3 道基础题
必须有基础掌握度规则
```

## C 级知识点门禁

```text
必须有教材来源
必须有基础说明
必须有知识图谱位置
必须有基础练习入口
```

## QuestionType 门禁

每个题型必须检查：

1. 题型定义
2. 适用知识点
3. 难度梯度
4. 常见错因
5. 验证题模板
6. 推荐策略包

## ErrorTaxonomy 门禁

每类错因必须检查：

1. 定义
2. 触发条件
3. 反证条件
4. 验证动作
5. 对应策略包
6. 家长端中性解释

## StrategyPack 门禁

每个策略包必须检查：

1. 策略名称
2. 适用错因
3. 适用学段
4. 适用题型
5. 执行动作
6. 话术模板
7. 结束条件
8. 效能记录字段
9. 禁用条件

## ParentSummaryTemplate 门禁

每个家长摘要模板必须检查：

1. 是否使用中性表达
2. 是否避免禁止词
3. 是否包含具体现象
4. 是否包含系统支持动作
5. 是否包含积极变化或下一步建议
6. 是否不展示原始错因概率
7. 是否不做心理判断

## 发布规则

### blocker
阻断发布。

包括：A 级知识点缺必要字段、错因无定义、策略无适用场景、家长摘要模板包含风险词。

### warning
允许内部测试，但不建议正式发布。

### pass
通过。
