## 芽芽助教项目背景

芽芽助教是一个面向 K12 全学段、多学科场景的自我进化 AI 教学适配引擎。它以真实教材知识图谱为骨架、以学习行为为真值、以错因诊断为核心、以策略调度为动作系统、以成长记忆为长期资产。

一句话：孩子用得越久，芽芽越懂他；芽芽教得越久，越知道怎么教他。

**当前建设范围**：人教版小学数学 1-6 年级全量覆盖。

**数据底座**：
- 394 个知识点（G1-G4: 205 个，G5-G6: 189 个），全量录入
- 86 个错因（L1 通用 10 个 + L2 数学特有 76 个），全量定义
- 1390 条知识点-错因绑定，394 个 KP 全覆盖
- 两层知识图谱（学习路径层 + 诊断依赖层），G1-G6 全部建成
- 题型规格矩阵（260KB）
- 数据质量门禁 data_quality_gate.py（9 铁规全覆盖）

**引擎架构**：诊断引擎（四步：初筛→匹配→验证→溯源）→ 策略引擎（六态调度）→ 家长摘要（中性安全反馈）→ 质量仪表盘（24 指标）。

**文档集**：14 份规划文档（01-14），覆盖产品愿景、开发路线、教材来源、分级规则、覆盖门禁、知识图谱、错因分类、策略包、事件协议、数据模型、诊断引擎、策略引擎、家长摘要、质量仪表盘。

## 本文档背景

这是芽芽助教产品文档集的第 08 号文件，定义 40 个策略包（通用+数学专属+学段话术模板），含编码体系、适用条件、互斥规则、效能评估和历史有效率计算。它回答「芽芽该怎么教」——诊断引擎定位错因后，策略引擎从本文档的策略池中选择对应动作。本文档是策略引擎（12 号文件）的数据基础。

## 一、策略包体系

策略包是引擎执行的"标准动作"。每个策略包有唯一编码、适用条件、执行动作、效果评估。

三层体系：

| 层级 | 说明 | 示例 |
|------|------|------|
| 通用策略包 | 跨学科通用的讲解/练习/反馈模式 | 基础巩固包、分步引航包 |
| 学科策略包 | 小学数学特有的图示/计算/建模包 | 分数图示包、小数点定位包 |
| 学段话术模板 | 不同年级用不同语言表达同一策略 | 低年级"我们一起数一数" vs 高年级"用小数位数逆推" |

映射规则：

- 数学专属策略包（SP_M_*）的触发条件通过本映射表由错因决定，不由第四节「适用知识点领域」列驱动。第四节的知识点领域仅供人工理解和教研参考——引擎始终以错因诊断结果（来自 11_DIAGNOSIS_ENGINE.md）为输入，查本映射表得策略包编码

## 六、策略效能评估

每次策略执行后（strategy_completed 事件），评估 5 个维度：

| 维度 | 评估方式 | 权重 |
|------|----------|------|
| 当下有效性 | 策略执行后下一道同类题是否正确 | 0.25 |
| 短期持续性 | 当次课堂内同类题正确率 | 0.25 |
| 长期保持性 | 7 天后再遇同类题的正确率 | 0.20 |
| 依赖降低 | 下次同类题是否需要相同提示 | 0.15 |
| 独立能力 | 无提示下同类题正确率 | 0.15 |

效能评分 effectScore ∈ [-2, 2]：

```
effectScore = Σ（维度得分 × 权重）

+2：显著提升（确认有效）
+1：有帮助
 0：无明显变化
-1：可能造成混淆
-2：造成退步或依赖（确认无效/有害）
```

效能数据写入 StrategyEffect 表（10_DATA_MODEL.md），后续策略选择时作为先验调整权重。

### 6.1 Phase 1 过渡规则

Phase 1 的事件协议不包含跨 session 长期追踪机制（如 7 天后复查），因此长期保持性维度在 Phase 1 无法采集数据。

Phase 1 临时权重分配：

| 维度 | Phase 1 权重 | Phase 2 权重 | 说明 |
|------|-------------|-------------|------|
| 当下有效性 | 0.30 | 0.25 | 上调 |
| 短期持续性 | 0.30 | 0.25 | 上调 |
| 长期保持性 | 0 | 0.20 | Phase 2 恢复 |
| 依赖降低 | 0.20 | 0.15 | 上调 |
| 独立能力 | 0.20 | 0.15 | 上调 |

Phase 2 上线后，需新增以下事件类型以支持长期保持性数据采集：

- `followup_assessment_scheduled`：7 天后安排复查题目
- `followup_question_answered`：复查题目作答结果

Phase 2 启动时，权重自动切换回标准配置，已写入的 Phase 1 effectScore 不重新计算（历史数据保持不变）。

### 6.2 家长反馈双信号机制

- effectScore 保持纯数据驱动，由系统自动计算，不被人为主观覆盖
- 家长反馈作为独立信号存储在 StrategyEffect 表的 `parentFeedback` 字段中（1=正面，0=中性，-1=负面）
- 策略引擎选包时同时读取两个信号：
  - effectScore 为负 且 parentFeedback 为负 → 该策略降为最低优先级（相当于禁用）
  - effectScore 为正 但 parentFeedback 为负 → 降一级优先级，不禁用
  - effectScore 为负 但 parentFeedback 为正面 → 保持原有优先级，不自动降权

这保证了客观数据的独立性，同时尊重家长反馈。

## 七、安全边界

### 7.1 强策略触发条件

以下策略包为"强策略"（变式特训包 SP_G_C02、错因专项包 SP_G_C03），仅当以下条件同时满足时才触发：

1. diagnosisStatus = confirmed（概率 ≥ 0.85）
2. 学生答卷数 ≥ 30（非冷启动）
3. 历史策略有效率 ≥ 0.5，且该策略包对该学生的使用次数 ≥ 3 次

其中，"历史策略有效率"定义为：该学生对该具体策略包（同一 strategyCode）的历史 effectScore 均值 ≥ 0.5，且该策略包已对该学生执行过 ≥ 3 次（少于 3 次时均值不稳定，不做判断）。不同策略包之间不混算。

**首次确诊例外**：该学生对该错因的首次确诊，跳过条件 3（历史策略有效率）。首次确诊后策略引擎执行该强策略 → 产生 StrategyEffect 记录 → 后续同一学生同一 strategyCode 再次被触发时，条件 3 生效。

### 7.2 中度策略

以下策略包为"中度策略"（精准计算包 SP_G_C05），触发条件：

1. 学生答卷数 ≥ 30（非冷启动）
2. 无需 diagnosisStatus = confirmed（可在低置信度时使用）

精准计算包不属于强策略的原因：纯计算练习即使做错了也不会产生概念混淆风险。最差情况是"练了但不是根因"，不会造成伤害。无需等待确诊即可执行。

### 7.3 轻量安全策略

低置信度、冷启动、连续错误时，只能触发轻量策略：

- 基础巩固包（SP_G_C01）
- 记忆唤醒包（SP_G_C06）
- 舒缓讲解包（SP_G_D01）
- 能量恢复包（SP_G_D03）
- 格式规范包（SP_G_B06）
- 检查清单包（SP_G_E04）

Phase 1 回退规则：轻量安全策略中若某策略包未在 Phase 1 实现（如 B06 格式规范包），引擎选中后回退到 SP_G_C01 基础巩固包。

### 7.4 策略滥用防护

| 防护规则 | 说明 |
|----------|------|
| 同一 lessonId 内同一策略 ≤ 3 次 | 防止疲劳。以 lesson_started 事件生成的 lessonId 为计算边界 |
| 同一 lessonId 内能量恢复包 ≤ 1 次 | 防止负向激励 |
| 连续使用同一策略超过 5 次时，触发 effectScore 衰减检测 | 比较最近 2 次该策略的 effectScore：若连续 2 次递减 → 强制变式；若持平或上升 → 继续使用，但每次追加衰减检测 |
| StrategyEffect = -2 且 parentFeedback = -1 的策略 7 天内禁用 | 数据+家长双重确认有害 |
| 家长反馈为负的策略降低优先级 | 尊重家长反馈，但不永久禁用 |
